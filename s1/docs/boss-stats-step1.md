# Boss 基础生存属性整理（第一步）

数据来源：
- `s1/data/war3map.w3u.json`
  - `uhpm` = 基础生命最大值
  - `uhpr` = 基础生命回复
- `s1/data/war3map.j`
  - 游戏面板显示的魔法防御来自运行时属性 `ds`
  - 游戏面板显示的伤害减免来自运行时属性 `fs`

说明：本次只整理基础单位数据。未发现这些 Boss 在单位 JSON 或 Boss 注册处直接写入固定 `ds/fs` 初始值，因此“基础魔防/基础减免”先按 `0%` 记录；技能、阶段、Buff 造成的临时变化后续再单独查。

## Boss 血量与人数关系

脚本中的实际最大生命函数为：

```text
u6_O = sQ + SQ + tQ
```

- `sQ`：单位基础生命，来自 `war3map.w3u.json` 的 `uhpm`
- `SQ`：常驻最大生命加成，人数缩放主要通过这里的最大生命百分比状态实现
- `tQ`：临时最大生命加成，例如技能、阶段、临时 Buff

人数缩放不是直接改 `uhpm`，而是根据 Boss 注册时绑定的缩放组，给 Boss 添加“最大生命 +X%”状态。基础公式：

```text
人数加成% = (clamp(人数, 最小人数, 最大人数) - 最小人数 + 1) * 每档加成%
实战最大生命 ≈ 基础生命 * (1 + 人数加成%)
```

人数低于最小人数时没有加成；人数高于最大人数时按最大人数封顶。

| 缩放组 | 人数范围 | 每档最大生命加成 | 常见 Boss 段 |
|---|---:|---:|---|
| `gr` | 无 | 0% | 部分早期/野外 Boss |
| `Gr` | 2-5人 | +50% | 粉末段、部分野外后段 |
| `hr` | 2-7人 | +40% | 小四 |
| `Hr` | 2-10人 | +35% | 四大、主龙 |
| `jr` | 4-10人 | +25% | 白怨火水、君主 |
| `Jr` | 5-10人 | +25% | 公爵 |
| `Kr` | 5-10人 | +30% | 土灵战神 |
| `Lr` | 4-6人 | +33.3333% | 古代魔导机器 |
| `mr` | 6-10人 | +30% | 灵魂收割者、大天使 |

例：`Gr` 组 Boss，基础生命 1,000,000。

| 人数 | 实战最大生命 |
|---:|---:|
| 1 | 1,000,000 |
| 2 | 1,500,000 |
| 3 | 2,000,000 |
| 4 | 2,500,000 |
| 5+ | 3,000,000 |

显示面板里的 `玩家数: p0/R8` 中，`p0` 是该 Boss 区域参与人数，`R8` 是全局有效玩家数。困难模式通过 `uu60` 额外修改伤害减免/掉落相关属性；当前未确认它会直接增加最大生命。
