# Boss 点击复活规则

来源：`s1/data/war3map.j` 的记忆石/复活商店逻辑，核心函数为 `iOpoi`、`iOp_i`、`iOp6i`。

## 机制

1. `iOpoi(bossId, count, reviveTime)` 注册某个 Boss 的点击复活规则。
2. `count` 是基础可复活次数，`reviveTime` 是点击复活后的基础等待时间，单位为秒。
3. 玩家在复活商店购买复活项时，`iOp_i` 会消耗 1 次次数并调用 `i_06(unit, reviveTime)` 执行复活等待。
4. 当前全局 `A8 = 1 - 0.01 * min(5 * (有效玩家数 - 1), 30)`，所以实际等待时间为 `reviveTime * A8`。
5. `iOp6i` 会在 Boss 死亡后按 `HP` 档保证部分 Boss 的可复活次数不低于目标值。
6. 没有 `iOpoi` 注册的 Boss，当前未发现记忆石点击复活规则。

## 次数规则

| Boss等级 | 适用分组 | 复活次数公式 |
| --- | --- | --- |
| `HP = 1` | 粉末、部分野外 | 15次 |
| `HP = 2` | 小四、部分粉末/野外 | 15次 |
| `HP = 3` | 四大、主龙 | 12次 |
| `HP = 4` | 白怨火水、君主 | 10次 |
| `HP = 5` | 爵土马判、大天使 | 8次 |

等待时间已按新规则减半。实际等待时间仍会吃全局多人修正，2人时为基础时间的 `95%`，最多降到基础时间的 `70%`。
